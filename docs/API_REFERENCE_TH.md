## [0] รายการเช็คลิสต์ก่อนเริ่มภารกิจ (MISSION CHECKLIST)
ควรทำตาม 4 ขั้นตอนนี้ทุกครั้งก่อนการรันจริงเพื่อให้หุ่นยนต์ทำงานได้อย่างแม่นยำที่สุด

1. **ตรวจสอบแบตเตอรี่ (Battery Check)**: แรงดันไฟฟ้าควรสูงกว่า 7.8V เพื่อให้ระบบ PID ทำงานได้คงที่
2. **การอุ่นเครื่องไจโร (Gyro Calibration)**: รัน `setup.full_setup()` และต้องวางหุ่นยนต์ให้ **นิ่งสนิท** 5 วินาที
3. **การคลิเบรตเซนเซอร์ (Sensor Calibration)**: ใช้ `CAL.sweep()` หรือ `setup.sensor_setup()` บนสนามจริง
4. **การจัดตำแหน่งจุดเริ่มต้น (Origin Alignment)**: ใช้ `wall_align()` หรือจัดด้วยมือให้ X, Y, H เป็น 0

---

## [1] การเริ่มต้นใช้งาน (GETTING STARTED)
```python
import setup, pid_lib as P

async def run_mission():
    # 1. ตรวจสอบสุขภาพแบตเตอรี่ (เตือนเท่านั้น)
    if not await setup.check_battery(warn_only=True):
        print("CRITICAL: Battery too low!")
        return

    # 2. เริ่มกระบวนการ Setup (Gyro + Sensors + Wall Align)
    odo = await setup.full_setup()
    
    if odo is not None:
        # 3. เริ่มทำภารกิจ
        await P.straight(odo, 50)
        await P.turn(odo, 90)
```

## [2] การเคลื่อนที่พื้นฐาน (BASIC MOTION)

### straight(odo, dist_cm)
เดินหน้า (+) หรือ ถอยหลัง (-) เป็นเส้นตรง
- **`odo`**: วัตถุ Odometry ที่ใช้ระบุพิกัดหุ่นยนต์ (หุ่นต้องรู้ตำแหน่งตัวเองก่อนเริ่มเดิน)
- **`dist_cm`**: ระยะทางที่ต้องการเดิน หน่วยเป็นเซนติเมตร (ค่าบวก = เดินหน้า, ค่าลบ = ถอยหลัง)

```yaml
Visual: |
      [  จุดหมาย  ]
      +----------+
      | [----]   |
      | |  O |   |
      | [----]   |
      +----------+
           /|\
            | (เดินตรง)
            |
      +----------+
      | [----]   |
      | |  O |   |
      | [----]   |
      +----------+
      [  จุดเริ่ม   ]
```

### turn(odo, target_h)
หมุนตัวรอบจุดศูนย์กลางไปยังองศาที่กำหนด (Point Turn)
- **`odo`**: วัตถุ Odometry
- **`target_h`**: องศาเป้าหมายในพิกัดโลก (-180 ถึง 180 องศา)

```yaml
Visual: |
      +----------+
      |  [ || ]  |
      |  [  O ]  | (หมุนไปที่ 90)
      |  [ || ]  |
      +----------+
           /|\
            | (หมุนอยู่กับที่)
            |
      +----------+
      |  [----]  |
      |  |  O |  | (เริ่มที่ 0)
      |  [----]  |
      +----------+
```

### pivot_turn(odo, target_h, side)
หมุนโดยล็อคล้อหนึ่งข้างและให้อีกข้างเคลื่อนที่
- **`odo`**: วัตถุ Odometry
- **`target_h`**: องศาเป้าหมายที่ต้องการให้หุ่นหันไป
- **`side`**: ล้อข้างที่จะให้หยุดนิ่งเป็นจุดหมุน เลือกได้คือ `'left'` (ล้อซ้าย), `'right'` (ล้อขวา) หรือ `'auto'` (เลือกทางที่สั้นที่สุด)

```yaml
Visual: |
               +------+
               | [||] |
               | [ O] |
               | [||] |
               +------+
              /
             / (แนวการสวิง)
            /
      +----------+
      | [FIXED]  |
      | |[X] |   |  <-- ล้อซ้ายอยู่กับที่
      | [----]   |
      +----------+
```

## [3] การเคลื่อนที่ด้วยเซนเซอร์ขั้นสูง (ADVANCED SENSOR MOTION)

### lf_n_junctions(n, vmax)
เดินตามเส้นและหยุดที่ "ทางแยก" (Intersection) ลำดับที่ N
- **`n`**: จำนวนทางแยกที่ต้องการจะข้ามไปก่อนจะหยุด
- **`vmax`**: ความเร็วสูงสุดในการเดินตามเส้น (0 ถึง 1000)

```yaml
Visual: |
      [  หยุดที่แยก #N ]
      +----------+
      |  [----]  |
      |  |  O |  |
      |  [----]  |
      +----------+
    -------+------- (แยกที่ 2)
           |
    -------+------- (แยกที่ 1)
           |
      [  จุดเริ่ม   ]
```

### lf_until_color(target_color, vmax)
เดินตามเส้นไปจนกว่าจะตรวจพบสีที่กำหนด
- **`target_color`**: ค่าคงที่ของสีที่ต้องการให้หยุด (เช่น `S.RED`, `S.BLUE`)
- **`vmax`**: ความเร็วสูงสุด (0 ถึง 1000)

```yaml
Visual: |
      [  หยุด  ]
      +--------+
      | [----] |
      | |  O | |  <-- หุ่นจอดบนสีเป้าหมาย
      | [----] |
      +--------+
      ==========  (โซนสีเป้าหมาย)
          |
          | (เดินตามเส้น)
          |
      +--------+
      | [----] |
      | |  O | |
      | [----] |
      +--------+
      [ เริ่มต้น ]
```

### detect_color_sequence(n_colors)
วิ่งผ่านและบันทึกลำดับของสีที่ตรวจพบ
- **`n_colors`**: จำนวนแผ่นสีที่ต้องการให้อ่าน
- **คืนค่า (Return)**: ลำดับของสีที่ตรวจพบในรูปแบบของ List (เช่น `[RED, BLUE, GREEN]`)

## [4] การควบคุมอุปกรณ์เสริม (ATTACHMENT CONTROL)

### motor_run_until_stall(p, speed)
หมุนมอเตอร์จนกว่าจะเจอแรงต้าน (เช่น การหนีบวัตถุหรือติดขอบ)
- **`p`**: พอร์ตของมอเตอร์ (เช่น `C.PORT_C1`)
- **`speed`**: กำลังของมอเตอร์ (ค่าบวกหรือลบตามทิศทางที่ต้องการหมุน)

### motor_home(p)
รีเซ็ตตำแหน่งแขนยกให้เป็น 0 โดยการหมุนจนติดขอบล่าง
- **`p`**: พอร์ตของมอเตอร์

## [5] เครื่องมือช่วยจัดตำแหน่ง (ALIGNMENT TOOLS)

### wall_align(speed)
ถอยหลังกระแทกผนังเพื่อให้หุ่นยนต์ตั้งตรงขนานกับขอบสนาม
- **`speed`**: ความเร็วในการถอย (แนะนำ 200-300)

### align_to_wall_color(target_color)
วิ่งเข้าหาจนกว่าเซนเซอร์ทั้งสองจะเจอโซนสี เพื่อจัดระนาบให้ตรงกับโซนสีนั้น
- **`target_color`**: สีเป้าหมายที่ต้องการจัดระนาบ
